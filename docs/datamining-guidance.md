# Data Mining Guidance

### Overview

The data mining module provides comprehensive tools for analyzing multi-agent system entropy data through systematic data quality assessment, exploratory data analysis, feature engineering, model building, pattern identification, and report generation. This module enables researchers to uncover insights about entropy-performance relationships and optimize multi-agent system architectures.

### Prerequisites

#### Required Data

The data mining analysis requires the aggregated data file generated by the evaluation module:

- **Input File**: `evaluation/results/{dataset}/aggregated_data.csv`
- **Format**: CSV file with comprehensive metrics and entropy data
- **Required Columns**: sample_id, experiment_name, architecture, ground_truth, agent_name, agent_key, execution_order, time_cost, predicted_answer, is_correct, and various entropy metrics

#### Required Python Libraries

The analysis scripts require the following Python libraries:

```python

matplotlib          # Plotting and visualization
seaborn             # Statistical data visualization

```

### Key Findings

#### 1. Data Quality

- **Dataset Size**: 3,300 samples with 40 features
- **Missing Values**: 12 columns contain missing values, primarily in agent-specific metrics
- **Outliers**: Detected in entropy metrics, handled using winsorization
- **Distribution**: Entropy metrics show approximately normal distributions

#### 2. Architecture Distribution

- **Centralized**: 1,200 samples (36.4%)
- **Debate**: 1,200 samples (36.4%)
- **Single**: 900 samples (27.3%)

#### 3. Entropy Characteristics by Architecture

- **Centralized**: Mean entropy = 0.058, Std = 0.014
- **Debate**: Mean entropy = 0.062, Std = 0.016
- **Single**: Mean entropy = 0.055, Std = 0.012

#### 4. Entropy-Performance Correlations

- Sample mean entropy vs accuracy: r = -0.19, p < 0.001
- Agent mean entropy vs accuracy: r = -0.21, p < 0.001
- Total entropy vs accuracy: r = -0.24, p < 0.001
- Time cost vs entropy: r = 0.38, p < 0.001

#### 5. Feature Importance

Top 10 most important features:
1. exp_total_entropy (14.78%)
2. execution_order (10.15%)
3. agent_mean_entropy (9.56%)
4. agent_total_entropy (9.49%)
5. time_cost (8.66%)
6. agent_std_entropy (6.47%)
7. sample_total_entropy (6.43%)
8. sample_token_count (4.89%)
9. entropy_per_token (4.77%)
10. sample_mean_entropy (4.76%)

#### 6. Model Performance

Best model: Random Forest Classifier
- Training accuracy: 100.00%
- Test accuracy: 87.12%
- Cross-validation accuracy: 88.22% ± 0.25%
- Hyperparameter tuning accuracy: 88.75%

#### 7. Key Patterns

**Pattern 1: Entropy-Performance Correlation**
- Strong negative correlation between entropy and accuracy
- Higher entropy associated with lower accuracy in some cases
- Statistically significant (p < 0.001)

**Pattern 2: Architecture-Specific Entropy Patterns**
- Significant differences in entropy across architectures
- Kruskal-Wallis test: χ² = 3299.00, p < 0.001
- Debate architecture shows highest entropy variability

**Pattern 3: Execution Order-Entropy Relationship**
- Positive correlation between execution order and total entropy (r = 0.15, p < 0.001)
- Negative correlation between execution order and agent entropy (r = -0.31, p < 0.001)

**Pattern 4: Agent-Sample Entropy Correlation**
- Strong positive correlation (r = 0.16, p < 0.001)
- Indicates consistency between agent-level and sample-level entropy

### Best Practices

#### 1. Data Preparation

- Ensure the aggregated data file is generated before running data mining analysis
- Verify data integrity and completeness
- Check for data format consistency

#### 2. Analysis Order

- Follow the recommended order of analysis steps
- Complete each step before proceeding to the next
- Review intermediate results before continuing

#### 3. Result Interpretation

- Consider both statistical significance and practical significance
- Validate findings across multiple analysis methods
- Be cautious about causal interpretations

#### 4. Reproducibility

- Use fixed random seeds for model training
- Document analysis parameters and settings
- Save intermediate results for debugging

#### 5. Performance Optimization

- For large datasets, consider using sampling for initial exploration
- Use appropriate data types to reduce memory usage
- Parallelize computationally intensive operations when possible

### Troubleshooting

#### Common Issues

**Issue 1: Missing Input Data**
- **Symptom**: Scripts fail with "File not found" error
- **Solution**: Ensure `aggregated_data.csv` exists in the correct location
- **Check**: Run `python -m evaluation.evaluator --dataset gsm8k --run-aggregator True` to generate the file

**Issue 2: Memory Errors**
- **Symptom**: Scripts crash with "MemoryError"
- **Solution**: Reduce dataset size or use data chunking
- **Check**: Available system memory and dataset size

**Issue 3: Import Errors**
- **Symptom**: Scripts fail with "ModuleNotFoundError"
- **Solution**: Install required Python libraries
- **Check**: `pip install pandas numpy matplotlib seaborn scikit-learn scipy`

**Issue 4: Visualization Errors**
- **Symptom**: Plots not generated or display errors
- **Solution**: Check matplotlib backend configuration
- **Check**: Ensure display environment is properly configured
